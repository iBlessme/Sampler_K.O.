# K.O. Sampler

Веб-приложение сэмплера и грувбокса с шаг-секвенсором. Работает прямо в браузере без установки — на основе Web Audio API.

**Версия:** 1.0.8

---

## Возможности

### Пэды
- 16 пэдов для воспроизведения звуков
- Загрузка собственных аудиофайлов на любой пэд (MP3, WAV, OGG и др.)
- Если сэмпл не загружен — пэд играет встроенный синтезированный звук
- Переименование пэдов (до 8 символов)
- Визуальная индикация: подсветка при ударе, маркер наличия сэмпла, маркер наличия паттерна

### Банки
- 3 независимых банка **A / B / C** — по 16 пэдов в каждом (итого 48 слотов)
- Переключение банков не прерывает воспроизведение

### Шаг-секвенсор
- 16 шагов на бар, **4 бара** на трек (64 шага суммарно)
- Независимый паттерн для каждого из 16 пэдов
- Переключение между барами во время воспроизведения
- Запись в реальном времени (REC) — нажатые пэды фиксируются в текущий шаг
- Режимы: очистить бар, очистить всё, случайная генерация паттерна
- Копирование и вставка паттернов между пэдами

### Транспорт
- Play / Stop
- BPM от 40 до 220 (кнопки с удержанием для быстрой прокрутки, стрелки ↑↓ на клавиатуре)
- Индикаторы статуса: питание, запись, воспроизведение

### Глобальные эффекты (ноббы)
| Нобб | Описание |
|------|----------|
| VOL | Общая громкость |
| PITCH | Общий питч |
| ATK | Время атаки (для синтезированных звуков) |
| REV | Реверб (свёрточный, на основе случайного IR) |

### Режимы пэдов
- **CHP** (Chop) — режим нарезки
- **LOP** (Loop) — зацикливание сэмпла при воспроизведении
- **REV** — воспроизведение задом наперёд

### Настройки пэда (Pad Settings)
Для каждого пэда индивидуально:
- Громкость, питч, панорама (pan), время спада (decay)
- Root note (MIDI-нота, 0–127)
- Реверс, луп, мют, соло
- Просмотр формы волны загруженного сэмпла
- Прослушивание пэда прямо из настроек (кнопка ▶)
- Сброс к настройкам по умолчанию

### Библиотека сэмплов
- Встроенная библиотека пресетов (Built-in) — пак готовых звуков
- Локальная библиотека (Local) — подключение папки с файлами через File System Access API
- Загрузка целого кита одной кнопкой (LOAD KIT)
- Переподключение папки при повторном открытии

### Хранение данных
- Загруженные сэмплы сохраняются в **IndexedDB** — не слетают при перезагрузке страницы
- Состояние (паттерны, настройки пэдов, BPM, ноббы, банк) сохраняется в **localStorage**
- Восстановление полного сеанса при следующем открытии

### Полноэкранный режим
- Кнопка в хедере (десктоп/планшет) — разворачивает приложение на весь экран
- Работает через стандартный Fullscreen API

### VU-метр
- Отображает уровень громкости при каждом ударе
- Цветовая индикация: зелёный → оранжевый (peak) → красный (clip)

### Отображение волны (Wave Strip)
- Полоса под info-bar показывает форму волны активного сэмпла
- Для синтезированных пэдов — отображает форму осциллятора (пила, квадрат, треугольник, синус)

### Клавиатура
| Клавиша | Действие |
|---------|----------|
| Q W E R | Пэды 1–4 |
| A S D F | Пэды 5–8 |
| Z X C V | Пэды 9–12 |
| 1 2 3 4 | Пэды 13–16 |
| Пробел  | Play / Stop |
| ↑ / ↓   | BPM +/– |
| [ / ]   | Переключение баров |

Клавиатура работает независимо от языковой раскладки (используется физическая позиция клавиши).

## Запуск

Проект — статический сайт, для работы ES-модулей требуется локальный HTTP-сервер.

```bash
# Node.js
npx serve .

# Python
python -m http.server 8080
```

Затем открыть `http://localhost:8080` в браузере.

Либо открыть через встроенный сервер WebStorm / VS Code (Live Server).

## Структура проекта

```
index.html          — разметка
css/
  styles.css        — все стили
js/
  state.js          — константы и общее состояние
  toast.js          — уведомления
  audio.js          — Web Audio API: воспроизведение, синтез, VU, waveform
  pads.js           — сетка пэдов, long-press, hit
  sequencer.js      — шаг-секвенсор
  ui.js             — ноббы, транспорт, меню, настройки пэда, клавиатура
  library.js        — библиотека сэмплов (built-in + local)
  storage.js        — IndexedDB (аудио) + localStorage (состояние)
  main.js           — инициализация, восстановление сессии
```

## Адаптивность

- **Mobile portrait** — полноэкранный режим, пэды занимают основную часть экрана
- **Mobile landscape** — двухколоночная сетка, wave-strip скрыт
- **Tablet / Desktop** — секвенсор в боковой панели рядом с пэдами, увеличенный размер шрифтов, кнопка полноэкранного режима

---

## Технологии

- Vanilla JS (ES Modules, без зависимостей)
- Web Audio API — `AudioContext`, `BufferSource`, `StereoPanner`, `ConvolverNode`
- Canvas API — отображение формы волны
- IndexedDB — хранение аудиобуферов
- File System Access API — локальная библиотека сэмплов
- CSS Grid + `clamp()` для адаптивной вёрстки
